# Альфа Бизнес AI

Демонстрационный прототип финансового помощника для малого бизнеса. Приложение загружает банковскую выгрузку, показывает ключевые показатели, выявляет риски, строит прогноз выручки и сравнивает демонстрационные тарифы.

> Все тарифы и финансовые условия в прототипе являются демонстрационными и не относятся к действующим продуктам Альфа-Банка.

## Возможности

- дашборд с выручкой, расходами, прибылью и средним чеком;
- аналитика операций с фильтрами и разбивкой по категориям;
- прогноз выручки на три месяца в трех сценариях;
- предупреждения об аномалиях и риске кассового разрыва;
- сравнение демонстрационных тарифов и расчет возможной экономии;
- ИИ-помощник, работающий в mock-режиме без API-ключа.

## Архитектура

`pages` отвечает за Streamlit-интерфейс, `services` содержит бизнес-логику, `repositories` изолирует SQLAlchemy, `schemas` описывает структуры данных, а `utils` содержит загрузку, проверку и форматирование. SQLite создается локально при первом запуске.

## Структура

```text
app.py                  # точка входа Streamlit
config.py               # переменные окружения
database.py, models.py  # база данных и ORM
pages/                  # экраны приложения
services/               # расчеты и сценарии
repositories/           # доступ к данным
schemas/                # dataclass/TypedDict-схемы
utils/                  # загрузка, валидация, форматирование
data/                   # демонстрационные CSV
tests/                  # модульные тесты
```

## Стек

Python 3.12, Streamlit, pandas, NumPy, Plotly, SQLAlchemy, scikit-learn, python-dotenv и pytest.

## Установка и запуск

Windows PowerShell:

```powershell
cd alfa_championship
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

Linux/macOS:

```bash
cd alfa_championship
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

Тесты:

```bash
pytest
```

## Формат CSV

Обязательные колонки: `date`, `amount`, `operation_type`, `description`. Дополнительно поддерживаются `category` и `payment_channel`. Допустимые типы операций: `income`, `expense`, `refund`, `commission`. Дата должна распознаваться pandas, сумма должна быть неотрицательной.

Демонстрационный сценарий использует вымышленный интернет-магазин одежды за шесть месяцев: рост продаж, всплеск рекламы, возвраты, крупная закупка и приближение к лимиту переводов. Реальные банковские данные, действующие тарифы и персональные сведения в репозитории отсутствуют.

## Ограничения прототипа

Прогноз основан на простой линейной модели и не является финансовой гарантией. Рекомендации помощника не заменяют бухгалтерскую, налоговую или юридическую консультацию. Подключение реального LLM оставлено точкой расширения: без `LLM_API_KEY` приложение отвечает из рассчитанного контекста.
